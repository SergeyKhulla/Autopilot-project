## О проекте

Это небольшой проект неполноценного автопилота, где я пытался освоить основы робототехники и соединить физику с программированием. Тут представлена простая симуляция автопилота и логика для Arduino контроллера.

Полное описание здесь:
https://docs.google.com/document/d/1GMn2BQZU7wF96sRZ0T99AP8iQmtZ72mOpWAY7Uo9B7o/edit?usp=sharing

#### Идея была такая

сначала сделать логику и тесты в виртуальной среде (Jsbsim + python pid), далее занести на физический контроллер и протестировать на реальном самодельном самолете (игрушечном, из пенопласта). Соответствующие модули в проекте arduino/ и python/

#### Внутри

arduino/autopilot/

	autopilot - входная точка, соединение логики всех модулей + логи
	config - важные переменные, чтобы было удобно делать дебаг
	pid - логика самого pid-контроллера
	sensors - взаимодействие с сенсорами
	servos - взаимодействие с серво и двигателем

python/

	altitude_controller - соединение pid и симулятора (изначально проект был как "удержание высоты", но я пока не сделал ничего больше, поэтому менять не стал)
	jsbsim_runner - запуск jsb тестов и логи
	pid - python версия pid
	tests/ - легкие тесты на проверку адекватности кода
	scenarios/ - yaml сценарии для jsb
	plots/ - граф отчеты для лучшего понимания происходящего

+всякие системные данные для jsb (вы можете найти там и конфигурацию самолета danala - ту, что я строю потихоньку, но она не используется, т.к. жутко неточная)

#### Что было сделано

Программная часть: 
Выбрал фреймворк jsbsim как достаточно простой, чтобы разобраться самому, но с более-менее достоверной физикой. Написан pid регулятор для крена и тангажа (для тангажа он двухконтурный требуемая высота - требуемый тангаж)

Аппаратная часть:
Вообще по-хорошему надо сделать летающий прототип, но пока я успел только модель-стойку для демонстрации. На ней представлен весь контур управления и датчиков (3 серво, двигатель, 6-осевой акселерометр, магнетометр, барометр, esc-контроллер и аккумулятор)

#### Что осталось

- Сделать более точную модель для jsb, которая отражает физику именно моего самолета, а не предустановленного
- Подобрать точные коэффициенты для pid
- Достроить реальную летающую модель и протестировать на ней
- В перспективе добавить gps и реализовать настоящие полетные задания (сейчас, по всем известным причинам я это протестировать не могу) 

#### Как запустить

Для python части лучше сначала поставить зависимости из корня проекта:

```bash
python -m pip install -r requirements.txt
```

Проверить базовую логику pid и контроллера можно так:

```bash
python -m pytest python/tests
```

JSBSim сценарии лежат в `python/scenarios/`. Запустить один сценарий:

```bash
python python/jsbsim_runner.py python/scenarios/03_pitch_hold.yaml
```

Запустить сразу все сценарии и сохранить графики:

```bash
python python/jsbsim_runner.py python/scenarios --plot-dir python/jsb_plots
```

Arduino часть собирается отдельно через PlatformIO, это не связано с `requirements.txt`:

```bash
pio run
```
